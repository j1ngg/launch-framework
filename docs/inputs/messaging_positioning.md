# Messaging & Positioning

## Positioning

| Context | Positioning Statement |
|---------|----------------------|
| In the market | The first Claude Code project purpose-built for developer tools marketing. It turns your terminal into a marketing workbench that reads your product context and enforces technical writing standards across every asset. |
| Against SaaS AI writing tools (Jasper, Copy.ai) | SaaS AI writing tools produce content that sounds like consumer marketing. Tech Marketing Framework lives in your terminal, reads your product brief, and generates content calibrated for developers. No browser. No per-seat pricing. No marketing fluff in the output. |
| Against generic Claude Code skill packs | Generic skill packs are prompt collections with no domain expertise. Tech Marketing Framework embeds opinionated content guidelines, product context ingestion, and interconnected skills that reference each other. The difference between a bag of prompts and a workbench. |
| Against ChatGPT / Claude chat | Chat requires you to reexplain your product every session. Tech Marketing Framework loads your product brief, personas, competitive intel, and messaging automatically. Persistent context means consistent output. |
| Against manual process | Manual content creation takes 10x longer and the quality varies with the writer's energy level. The framework produces first drafts in minutes that meet a consistent quality bar, then humans edit. |
| Against agencies / freelancers | Agencies cost $5K to $20K per launch and rarely understand developer audiences. The framework costs $0 (plus your Claude subscription) and is calibrated specifically for technical buyers. |

## Market Category
**Claude Code for marketing.** This positions the product as part of the growing Claude Code ecosystem while claiming the marketing vertical. Prospects who search for "Claude Code projects" or "AI marketing tools" should find this.

The default category assumption for AI marketing tools is "AI content generator" (Jasper, Copy.ai). That category is saturated and carries negative connotations with technical audiences. By framing as a "Claude Code project" instead, the product inherits the credibility of the developer tools ecosystem rather than the baggage of the AI writing tools market.

### Language Rules

| Use | Never Use |
|-----|-----------|
| workbench | platform, solution |
| skills | features, modules |
| generates | creates, produces |
| technical audiences | customers, users (in marketing copy) |
| developer tools companies | SaaS companies, startups (be specific) |
| open source | free tool (open source signals trust; free signals cheap) |
| content guidelines | guardrails, safety rails |
| reads your product docs | understands your product (it reads, it does not understand) |

### Anti-Persona

Who is NOT a fit (so marketing and sales can disqualify early):

- Marketers who do not use the terminal
- Enterprise teams needing team collaboration and approval workflows
- Consumer brand marketers
- Companies selling exclusively to non-technical buyers
- People looking for a plug-and-play tool with zero setup (this requires filling in docs)
